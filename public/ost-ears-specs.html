IDTipoSentencia EARSEstado
REQ-OST-001 UBIQUITOUS El sistema DEBERÁ asociar cada Oportunidad a exactamente un Proyecto, impidiendo la creación sin project_id. IMPL
REQ-OST-002 UBIQUITOUS El sistema DEBERÁ asignar a cada miembro exactamente uno de los roles admin, usuario o viewer. IMPL
REQ-OST-003 STATE MIENTRAS el rol sea viewer, DEBERÁ deshabilitar todos los controles de escritura y renderizarlos en modo solo lectura. PARCIAL
REQ-OST-004 STATE MIENTRAS el rol sea admin, DEBERÁ habilitar la gestión de miembros (invitar, cambiar rol, remover). IMPL

IDTipoSentencia EARSEstado
REQ-OST-007 EVENT CUANDO el sistema se inicialice, DEBERÁ establecer cliente Supabase autenticado con NEXT_PUBLIC_SUPABASE_URL y NEXT_PUBLIC_SUPABASE_ANON_KEY. IMPL
REQ-OST-008 UNWANTED SI Supabase no puede conectar (timeout > 5s o error de red), ENTONCES DEBERÁ mostrar bloqueo con mensaje de error y botón de reintento. IMPL

REQ-OST-010 EVENT CUANDO el usuario modifique un campo, DEBERÁ ejecutar UPDATE dentro de los 500 ms posteriores al fin de la edición (debounce). IMPL
REQ-OST-011 EVENT CUANDO Realtime esté activo, DEBERÁ suscribirse a postgres_changes en las 5 tablas de dominio para reflejar cambios de otras sesiones. IMPL
